Binary Search Tree

A BST is a binary tree in symmetric order

binary tree is either: empty, two disjoint binary tree(left and right)

Symmetric order: each node has a key and every node's key is larger than all keys in its left subtree, and smaller than all keys in its right subtree

Different from heap where it has every node larger than both its children

    Get/Search cost: Number of compares = 1 + depth of node
    Put/Insert cost: Number of compares = 1 + depth of node
    Tree shape depends on order of insertion

A BST is uniquely defined by its level-order traversal, a breadth-first traversal where the root is visited first, then all nodes at depth 1 (going from left to right), then all nodes at depth 2(going from left to right), and so forth

2-3 Tree

1 or 2 keys per node

    2-node: one key, 2 children
      left - children with smaller keys
      right - children with bigger keys
    3-node: two keys, 3 children
      left - children with smaller keys
      between - children with keys between 2 keys
      right - children with bigger keys

perfect balance: every path from root to null link has same length

symmetric order: inorder traversal yields keys in ascending order

Red-Black BST

Left-leaning red-black BSTs

represents a 2-3 tree as a BST, where 3 nodes are 2 key is split, between them is red

most operation i.e. search is identical to BST

      No node has 2 red links connected
      Every path from root to null link has the same number of black links
      Red Links lean left

Summary

BST becomes unsymmetrical after insert and delete because

hilbert deletion finds successor from the right, eventually goes to sqrt N, instead of log N
